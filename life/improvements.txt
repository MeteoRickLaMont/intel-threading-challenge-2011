- jmfernandez had the right idea: A* algorithm (except without closed set).
- Board representation from nblock.c except fixed height and width. Do not
  skip over empty areas.
- File load is bottleneck. Custom algorithm to parse positive integers quickly.
- Exit immediately when solution is found. Don't wait for threads to join.
- Tune number of threads to size of problem. Singlethread for very small.
- Spawn thread to start other threads while main is parsing file, etc.
- No mutexes. Use atomics and lock free concurrent_priority_queue<>
- Intentionally leak resources.
- Store board + move in priority queue. Apply move and next gen upon pop.
  Advantages:
    Seeds priority queue faster, especially from initial position.
    Detect winning move without push/pop of priority queue.
- Prevent false sharing by putting each thread's local data in a separate
  cache line.
- Use TBB scalable allocator
